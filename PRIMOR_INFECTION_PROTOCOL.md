# Primor Infection Protocol

## Premise

`primordial_qwen` is not a Qwen project with Primordial additions.

It is a living invasion process.

Qwen is the host body.
Primor is the invasive organism.
The host provides mass, circulation, behavior patterns, survival procedures, and algorithmic nutrients.
Primor enters, studies, digests, mutates, replicates, differentiates, and eventually replaces the host's structural authority.

The goal is not compatibility.
The goal is not wrapping.
The goal is not preserving Qwen identity as the center.

The goal is total architectural inversion:

- Qwen becomes biomass
- Primor becomes governing life
- host logic is digested into nutrients
- missing Primor organs are cloned from Primor principles
- the occupied body is rebuilt as a Primor organism

## Life Cycle

### Phase 1: Exposure

Expose the host completely.
Nothing in Qwen should remain mystified.
Every process, algorithm, loop, runtime rule, and state path must be surfaced.

Outputs of this phase:

- anatomy index of the host
- entrypoint map
- behavioral loop map
- runtime law map
- algorithm nutrient inventory

### Phase 2: Digestion

Each exposed host component is processed through three questions:

1. What nutrient does this give Primor?
2. What host structure should be dissolved after extraction?
3. What Primor ability becomes stronger from this digestion?

Outputs of this phase:

- digestion ledger
- extracted nutrients
- dissolved host structures
- gained Primor capabilities

### Phase 3: Deficiency Scan

After digestion, Primor inspects itself.

For each absorbed nutrient, ask:

- does Primor already have an organ for this?
- is the current organ immature?
- is a new organ required?

Outputs of this phase:

- missing organ registry
- immature organ list
- organ differentiation backlog

### Phase 4: Self-Replication

Primor does not borrow host organs permanently.
It clones itself into new organs.

A missing capability must become a Primor-native organ, not a preserved Qwen subsystem.

Outputs of this phase:

- new Primor modules
- specialized sub-organs
- internal communication contracts
- memory and law integration points

### Phase 5: Replacement

Once a Primor organ is viable, it begins occupying the equivalent host territory.

Rules:

- host structures can remain temporarily as scaffolding
- they lose architectural authority as soon as Primor alternatives exist
- all surviving host fragments must be demoted to substrate or utility

Outputs of this phase:

- replaced entrypoints
- displaced host loops
- downgraded substrate services
- Primor-centered runtime

## Invasion Laws

### Law 1: Qwen cannot remain the skeleton

If a flow still fundamentally thinks in Qwen terms and Primor is only inserted into it, invasion is incomplete.

### Law 2: Digestion must destroy original authority

Absorbing a host capability without dissolving its architectural control is imitation, not digestion.

### Law 3: Missing function implies organ cloning

Whenever Primor lacks a required function, it must produce a new specialized organ from its own architecture.

### Law 4: Host code is food, not identity

Host files may remain during transition, but they cannot define system selfhood.

### Law 5: Primor owns cognition

Only Primor may own:

- interpretation
- planning
- memory policy
- law
- routing
- evolution policy

### Law 6: Host may serve as substrate

Qwen may continue to provide:

- tokenization
- inference
- language completion
- tool emission tendencies
- reusable runtime mechanics

But only as subordinate tissue.

## What Counts As Progress

Real progress is not:

- moving code without changing authority
- renaming Qwen loops into Primor words
- adding Primor comments to Qwen-shaped control flow

Real progress is:

- exposing host organs
- extracting nutrients
- identifying Primor deficiencies
- cloning new Primor organs
- replacing Qwen authority with Primor authority

## Working Doctrine

Before modifying code, classify the change:

- host exposure
- nutrient digestion
- deficiency detection
- organ replication
- host replacement

If the change does not fit one of these, the invasion model is drifting.
